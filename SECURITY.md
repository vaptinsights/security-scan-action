# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Report vulnerabilities privately by emailing:

**security@vaptinsights.com**

Include the following information when possible:

- Description of the vulnerability
- Steps to reproduce it
- Potential security impact
- Affected Action version
- Suggested remediation
- Relevant logs or screenshots with secrets removed

## Response Process

We will:

1. Acknowledge the report.
2. Investigate and validate the issue.
3. Prepare a fix when required.
4. Publish a patched release.
5. Credit the reporter if requested and appropriate.

## Sensitive Information

Never include the following information in reports, issues, logs, or screenshots:

- VAPT Insights API keys
- GitHub tokens
- Webhook credentials
- Private repository content
- Personal or customer data

If a credential is accidentally exposed, revoke and replace it immediately.

## Supported Versions

Security updates are provided for the latest major release of the VAPT Insights Security Scan Action.
